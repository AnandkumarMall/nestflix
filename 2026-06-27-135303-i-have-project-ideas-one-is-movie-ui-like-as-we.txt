  a player that streams the local file.

  /movie/{id}/recommendations and /trending endpoints, so you can show "Because
  you have X" rows and a "New in theaters" row right alongside your local
  library.

  Suggested stack

  A web app is the natural fit for a Netflix-like UI (and it works great running
  locally):

  - Backend: Python + FastAPI — handles folder scanning, TMDB calls, and

  nicely since your machine already looks Python-oriented.
  - Frontend: React + Vite for the grid/rows/player UI. Or, to keep it dead
  simple to start, plain HTML+JS served by FastAPI.
  - Storage: SQLite for the movie metadata cache.
  - Player: the browser's native <video> tag handles .mp4 well; .mkv is
  hit-or-miss in browsers — for full format support you'd transcode with ffmpeg

     backdrop, overview, rating, genres, cast, keywords, runtime. Cache
